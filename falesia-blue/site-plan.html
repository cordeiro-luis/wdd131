Overview

Purpose

FalésiaBlue provides users with an elegant, responsive dashboard to check coastal conditions (simulated water temperature, wave height, and wind safety) across Portugal's most stunning beaches and natural swimming pools. It showcases the author personal passion for the ocean and the outdoors, while also serving as a practical tool for both locals and tourists to make informed decisions about their beach visits.

Audience

The target audience includes both local Portuguese residents and international tourists/expats looking for the perfect, safe spot to swim, surf, or relax based on real-time ocean conditions.

Dynamic elements

  • The Data Array: An array of objects holding the name, region, type (Beach vs. Natural Pool), water temperature, wave height, and safety flag.
  • Array Methods:
    • On page initialization, .forEach() will loop through the array to automatically render the spot cards.
    • When a user clicks a region filter (e.g., "Algarve", "Arrábida", or "Azores"), .filter() will narrow down the displayed cards instantly.
  • DOM Interaction: JavaScript will listen for click events on the filter navigation and individual cards. Clicking a card will dynamically populate a "Daily Safety Verdict" banner at the top of the viewport, changing its text and background color dynamically.
  • Conditional Branching: The logic will analyze the location type. For example (to be fine-tuned based on actual data):
    • If (type === "Natural Pool" && waveHeight > 1.5) → Display warning: "High surf! Volcanic pools may be swept by waves. Stay clear."
    • Else if (waveHeight < 0.5) → Display: "Calm waters. Perfect for swimming and snorkeling!"

Pseudocode for your project

			
1. ON HOME PAGE LOAD:
   a. Select the "Today's Pick" container and the main grid container.
   b. Generate a random index or use a date-based logic to pick one "spot" from the array.
   c. Inject that spot's data into the "Today's Pick" hero banner (showing the location  image).
   d. Use .forEach on the entire 'spots' array to render the remaining beach cards below.
   e. Add a click event listener to all cards and the hero banner that saves the selected beach's name
      to LocalStorage, then redirects to 'beach-details.html'.

2. WHEN A FILTER BUTTON IS CLICKED (Home Page):
   a. Clear the main grid container.
   b. Run the .filter method on the 'spots' array based on the clicked button (e.g., "Azores").
   c. Use .forEach to render only the cards/locations that matched the filter.

3. ON DETAILS PAGE LOAD:
   a. Read the selected beach name stored in local storage (the objects array).
   b. Use the .find method on the 'spots' array to retrieve the complete object for that specific beach.
   c. Set the page background-image dynamically to the image URL of that specific beach.
   d. Inject the name, region, temperature, wind, and waves into the central card that will
      display the beach details. This card will be a bit transparent
	  so the background image is visible behind it.

4. EVALUATE SAFETY CONDITIONS (Details Page):
   a. Run an IF/ELSE statement on the selected beach data:
      - IF (type === "Natural Pool" AND waveHeight > 1.5) OR flag === "Red":
          Display Warning: "HIGH SURF! Volcanic pools may be swept by waves. Stay clear."
          Apply 'bg-danger' styling to the warning box.
      - ELSE IF (waveHeight < 0.5):
          Display Message: "CALM WATERS: Perfect for swimming and snorkeling."
          Apply 'bg-safe' styling to the warning box.
      - ELSE:
          Display Message: "CAUTION: Moderate conditions. Swim or surf with care."
          Apply 'bg-caution' styling to the warning box.

Wireframes

Create wireframes for your project page(s). One for each page and list them here.

Landing page

The landing page will feature a hero banner with a "Today's Pick" beach, followed by a grid of cards representing other beaches. A summary of the current conditions will be on the main pick. Each card will display the beach name, region, and a small picure. Filter buttons will allow users to narrow down the displayed beaches by region. The card jumps to the details page when clicked, where more information about the selected beach is displayed like water temperature, wave height, and safety warnings.

Page 2

The second page has details for each beach selected from the landing page. It will display a large background image of the selected beach, along with a central card that shows detailed information such as water temperature, wave height, wind conditions, and safety warnings. The page will also include a back button to return to the landing page. Or maybe a get directions button that opens Google Maps with the beach location.
